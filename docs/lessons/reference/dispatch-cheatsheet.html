Dispatch cheat-sheet

Compressed essence of lesson 0001. Source of truth: skills/nitpicker/SKILL.md → Dispatch.

The grammar

/nitpicker [command] [extra instructions]
            ^^^^^^^   ^^^^^^^^^^^^^^^^^^^^
            first     free text; scope/focus/thresholds.
            word      inline & changed-files may appear anywhere.

Resolution, in five lines

  1. First word → command candidate; the rest → extra instructions.
  2. Match the word against the SKILL.md tables — canonical or alias.
  3. No match, or empty → run default audit; all text becomes instructions.
  4. Read _conventions.md, then commands/<resolved>.md.
  5. Execute that one command. Router never self-chains.

Resolution cases

First word is…Result
a canonical name (perf)that command file
an alias (perf-auditor)the same command file
unknown, or absentdefault audit; text → instructions

Adding a command = two edits, enforced 1:1

DoEnforced by
Create commands/<name>.mdvalidate-skill.py — a file without a row, or a row without a file, fails CI
Add its row to a SKILL.md command table

Glossary

TermMeans
Routerskills/nitpicker/SKILL.md — the only file that dispatches; it parses the text and resolves one command.
Command filecommands/<name>.md — the full behaviour of one command. No frontmatter.
Canonical nameThe command's real name and filename stem (e.g. tests).
AliasAn alternate word (often a 1.x name like test-auditor) that resolves to the same command file.
Modifierinline (write nothing to disk) or changed-files (scope to modified files); may appear anywhere in the instructions.
_conventions.mdThe shared binding read before every command — severity, findings store, modifiers, rules.
Findings storedocs/audit/findings/<auditor>/, driven by findings.py. Where audit commands record results. (Non-audit commands like plan and teach override this.)
1:1 ruleEvery non-_ command file has exactly one SKILL.md table row and vice versa; _-prefixed files (shared references) are exempt.
