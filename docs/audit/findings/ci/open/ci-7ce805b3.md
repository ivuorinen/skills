# strict_required_status_checks_policy false allows merging a PR whose Validate ran against a stale base

## Problem

With strict mode off, a PR is mergeable on a green Validate even when main has advanced past its base.

## Evidence

`gh api repos/ivuorinen/skills/rules/branches/main` -> `"required_status_checks":{"strict_required_status_checks_policy":false,...}`.

## Impact

PR A bumps the version in package.json; PR B (opened earlier, Validate green against the old base) also touches a manifest. Both merge; main now fails check-version-sync.py even though every PR was green. required_linear_history is enabled, so this surfaces as a broken main rather than a merge conflict.

## Fix

Set strict_required_status_checks_policy: true on ruleset 15530545. Squash-only merging is already enforced, so the re-run cost is one run per stale PR.
