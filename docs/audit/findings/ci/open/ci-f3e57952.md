# The CODEOWNERS half of the VENDORED_SKILLS control is inert - code-owner review is off and the file is untracked

## Problem

vendored-skills.md states the allowlist control binds only while "Require review from Code Owners" is enabled and calls the branch-protection setting "part of the control, not an optional extra." It is off. Separately the file is not in git at all, so GitHub cannot resolve it from the branch tree.

## Evidence

`gh api repos/ivuorinen/skills/rules/branches/main` -> `"pull_request":{...,"require_code_owner_review":false,...}`. `git status --porcelain` -> `?? .github/CODEOWNERS`; `git ls-files .github/CODEOWNERS` returns empty. .claude/rules/commit-types.md is untracked by the same check, so validate-rules.py's "11 rule(s) validated" counts a rule that does not exist on main.

## Impact

An agent or contributor can add a skill name to VENDORED_SKILLS and update test_allowlist_contains_only_approved_entries in the same two-line diff - the rule's stated failure mode - with no code-owner review triggered. The same hole covers /.claude/agents/, /.claude/settings.json, /LICENSE and /NOTICE.

## Fix

`git add .github/CODEOWNERS .claude/rules/commit-types.md` and commit. Then enable require_code_owner_review on the main ruleset (id 15530545). Until both are done, vendored-skills.md overstates the control and should say so.
